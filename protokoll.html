The simple data protocol

The simple data protocol (sdp) is used for simple and fast data transportation.

Currently there are two different kinds of clients: DataProviders and DataListeners

DataListeners choose which DataProviders they want to receive data from and consequently get sent that data.

Every packet has the same header specifying version, sender and packettype.

Size [bytes] Description Name Possible values
1 specifies the protocol of subsequent packet ProtocolID 0xDD for sdp
2 version of sdp, first byte for major, second byte for minor version Version 1.0 for the version specified in this document
1 specifies the type of subsequent data Packettype specified further down
variable data of the actual package    

PacketTypes

Name Packettype Description
ClientConnect 1 client connection
ClientConnected 2 server response to client connection
DataProviderList 3 list of all data providers (by label)
ClientStartListen 4 client starts listening to a dataprovider
ClientListening 5 server response to ClientStartListen
Data 6 provides data to a datalistener from a dataprovider
DataProvide 7 provide data from a datalistener

Connecting To connect a client has to send a ClientConnect packet with its type and label:

ClientConnect:

Size [bytes] Name Description Possible values
1 Type Type of Client 0 for DataProvider, 1 for DataListener
7 Label Unique label of the client 6 character zero terminated string

This packet is responed by the server with a ClientConnected packet specifying the connection status:

ClientConnected:

Size [bytes] Name Description Possible values
1 ConnectionStatus A Connection can fail because of several reasons, the exact reason is indicated by the value 0 ConnectionSuccessful
      1 LabelNotUnique
      2 LabelNotZeroTerminated
      3 Another error occured

Gathering data After successful connection the server sends a DataProviderList packet, with the labels of all connected dataproviders.

DataProviderList:

Size [bytes] Name Description Possible values
7*n Label Array of the labels of all connected dataproviders array of zero terminated 6 character strings

To actually get data from a DataProvider a DataListener has to send a ClientStartListen packet with the label of the DataProvider he wants to get data from:

ClientStartListen:

Size [bytes] Name Description Possible Values
7 Label label of the DataProvider the client wants to listen to a zero terminated 6 character string

This packet is responded by the server with a ClientListening packet indicating the success of the ClientListen packet

Size [bytes] Name Description Possible Values
1 ListenStatus Gives the status of the listening 0 ListenSuccessful
      1 InvalidLabel
      2 Some other error occured

Actual Data: As soon as you are listening to some DataProviders you get their data in Data packet:

Data:

Size [bytes] Name Description Possible Values
8 Count Count of the data any unsigned 64 bit integer
variable Data The actual data anything

DataProviders can, as soon as they connected to the server, provide data via the DataProvide packet:

DataProvide:

Size [bytes] Name Description Possible Values
8 Count Count of the data any unsigned 64 bit integer
variable Data The actual data anything

The server does not cache the data, so any data provided by a DataProvider without a DataListener listening to that DataProvider is lost!
